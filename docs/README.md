# Documentation

This folder contains client-safe product documentation and internal acceptance/contract notes.

## Files
- `RELEASE_NOTES_v1.0_EN.md` — English release notes.
- `RELEASE_NOTES_v1.0_RO.md` — Romanian release notes.
- `VERSIONING_RULES_v1.0.md` — Versioning policy.
- `internal/PHASE3_ACCEPTANCE.md` — Phase 3 UI/workflow acceptance checklist (internal).
- `internal/PHASE4_RENDER_CONTRACT.md` — Render-only contract draft (internal).

## When to Update
Update these documents when:
- The delivery contract changes.
- Outcome semantics change (e.g., SUCCESS vs NOT AUDITABLE).
- A release is tagged.
